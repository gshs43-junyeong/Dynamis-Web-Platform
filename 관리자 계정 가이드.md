# 관리자 계정 가이드

Dynamis Web Platform 관리자(admin) 계정의 권한 범위와 관리자 콘솔 사용법, 그리고 최초 관리자 계정을 부여하는 절차를 안내합니다.

## 1. 관리자 등급 판정 방식

이 플랫폼은 별도 백엔드(Cloud Functions) 없이 운영되는 구조라, 관리자 여부는 Firebase Auth의 커스텀 클레임이 아니라 **Firestore `users/{uid}` 문서의 `role` 필드가 `admin`인지**로 판정합니다. (`firestore.rules`의 `isAdmin()` 함수 참고)

- 일반 가입은 항상 `role: 'general'`로 생성되며, 본인이 스스로 관리자로 승격할 수 있는 경로는 없습니다.
- 등급(`role`) 변경은 **이미 관리자인 계정만** 수행할 수 있습니다 (`isValidUserUpdate`의 `isAdmin || newData.role == oldData.role` 조건).

## 2. 최초 관리자 계정 부여 (부트스트래핑)

동아리에 관리자가 한 명도 없는 초기 상태에서는, 관리자 콘솔 자체를 아무도 열 수 없습니다. 최초 관리자는 **Firebase 콘솔에서 수동으로 지정**해야 합니다.

1. 지정하려는 사람이 먼저 사이트에서 일반 계정으로 회원가입을 완료합니다.
2. Firebase 콘솔 → Firestore Database → `users` 컬렉션에서 해당 사용자의 문서(문서 ID = 그 사람의 Auth UID)를 찾습니다.
3. `role` 필드 값을 `general`에서 `admin`으로 직접 수정합니다.
4. 해당 사용자가 재로그인(또는 새로고침 후 재인증)하면 관리자 콘솔(`/admin`)에 접근할 수 있습니다.

이후부터는 이 관리자가 콘솔의 등급 변경 기능으로 다른 관리자를 추가로 지정할 수 있습니다.

## 3. 관리자 콘솔 기능

관리자로 로그인하면 상단 네비게이션에 **관리자(Admin)** 메뉴가 나타납니다.

### 등급 변경
회원 목록에서 등급(`general`/`honored`/`member`/`admin`)을 드롭다운으로 선택하고 **변경 확정**을 누르면 즉시 반영됩니다.
- `general`: 등급 없음 (가입 직후 기본값)
- `honored`: 명예부원 (공지/이벤트 작성 가능, 파일 첨부 불가)
- `member`: 부원 (공지/이벤트 작성 및 파일 첨부 가능)
- `admin`: 관리자 (모든 권한 + 관리자 콘솔 접근)

### 경고 조치
**경고 조치** 버튼을 누르면 해당 계정의 누적 경고 횟수가 1회 증가하고, 다음 로그인 시 경고 알림이 표시됩니다. 경고 누적에 따른 자동 정지/삭제 처리는 없으며, 계정 강제 탈퇴 여부는 관리자가 직접 판단해 별도로 수행합니다.

### 계정 삭제 (강제 탈퇴)
**계정 삭제** 버튼은 대상 계정을 강제 탈퇴시킵니다.
- 대상이 작성한 공지사항/이벤트/댓글을 best-effort로 함께 정리합니다 (일부 정리가 실패해도 계정 삭제 자체는 진행됩니다).
- 대상의 일일 트래픽 통계는 규칙상 본인만 읽고 쓸 수 있어 관리자가 지울 수 없습니다.
- 대상의 Firebase Auth 로그인 계정 자체는 관리자가 삭제할 수 없습니다. Firestore 상의 계정 정보만 삭제되므로, 그 사람이 재로그인을 시도하면 "가입되지 않은 계정"으로 처리되어 재가입이 필요합니다.

### 공지사항 고정
공지사항 목록에서 관리자에게만 보이는 **고정** 체크박스로 특정 공지를 상단에 고정할 수 있습니다.

## 4. 주의사항

- 관리자 권한 남용(사적 목적의 정보 열람/유출 등)은 커뮤니티 이용 가이드라인 제4조에 의해 관리자 역할 박탈 및 계정 삭제 대상이 됩니다.
- `firestore.rules`를 수정한 뒤에는 반드시 Firebase 콘솔 → Firestore Database → 규칙에 재배포해야 변경 사항이 실제로 적용됩니다.
